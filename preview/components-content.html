Content

Two content primitives for a dev-tool surface — keyed to components.css. The code block carries a filename, a language tag, and a copy button (preview/code-copy.js, zero-dependency, sandbox-safe); its syntax colors are semantic and stay fixed. The metric tiles add a trend delta + optional sparkline on top of the plain .stat.

// Code block (filename + language tag + copy · line numbers via .code-lines)

// Source basis — the website's real build-time modules (onebrain-ai/website)
version.ts — the CLI version shown in a metric/badge; from the Releases API at prerender, falls back to 3.0.0. build
gh-stars.ts — the star-count metric; stargazers_count baked at build, hidden on 0. build
disposable-domains.ts — waitlist guard; a blocklist match returns 200 with no write. api
brain-id.ts — stable SSR gradient ids via a module counter (no Math.random). ssr
middleware.ts — HSTS / nosniff / DENY / COOP headers wrapped so they ship even on a 500. edge

Classes: .code-block / .code-head / .code-lines · .metric / .metric-trend in components.css §17–§18. Copy enhancer: preview/code-copy.js (reads via innerText, Clipboard API with an execCommand fallback for sandboxed contexts). Syntax token colors (.tok-key / .tok-str / .tok-fn) are semantic and fixed; the language tag + copy hover follow the section accent. Sparkline keyed to --chart-1.
