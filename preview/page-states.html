Page states

Full-page system states for the operator console — not-found, harness fault, access denied, vault offline, and the boot/loading screen. Each is a .page-state (components.css §12.5): a corner-cut mono status code, an italic Chakra headline, a mono sub-line, real .btn-tech / .btn-ghost actions, and a pulsing status line. Re-tint per state by setting one var — --section-accent.

Source basis · onebrain-ai/website
  • middleware.ts — the edge layer that emits these states: route guarding (404 / 403 redirects) and the security headers stamped on every response — Strict-Transport-Security, X-Content-Type-Options: nosniff, X-Frame-Options: DENY, Referrer-Policy: strict-origin-when-cross-origin.
  • brain-id.ts — the deterministic, SSR-safe request/session id (no Math.random) stamped on the 500 card (#brain-7f3a-c2) so a fault is traceable to a request.
  • version.ts — resolves the build version with a FALLBACK of 3.0.0; shown as the BUILD · v3.0.0 stamp on the boot screen.
  • disposable-domains.ts — the waitlist "silent-200" soft-reject (accept the POST, drop the address, never reveal the block) — the same graceful, non-alarming posture the offline state takes instead of a hard error.
  • gh-stars.ts — formatStars() (hide-on-0, 1200 → 1.2k) keeps the star chip rendering in the degraded offline shell, demonstrating graceful UI under data loss.
