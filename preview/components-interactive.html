version.ts · build-time CLI version
Fetches the latest version from the GitHub Releases API at prerender (no runtime cost), memoized so Hero and Footer share one in-flight fetch. Falls back to 3.0.0 on an offline / rate-limited build.
gh-stars.ts · star count, baked at build
Reads stargazers_count for onebrain-ai/onebrain at build. Returns 0 on any failure and the Cloud badge hides on 0 — "0 stars" as social proof is worse than none.
disposable-domains.ts · waitlist email guard
A ~150-domain inline blocklist. On a match the server returns 200 OK with no D1 write, so a bot can't distinguish "blocked" from "accepted" and stops retrying — protecting sender reputation.
brain-id.ts · stable SSR gradient ids
A module-scope counter (nextBrainId() => ++counter) gives two <Brain/> on a page distinct gradient/filter ids without Math.random — which would byte-shift the HTML every request and break edge caching.
middleware.ts · API security headers
Adds HSTS, X-Content-Type-Options: nosniff, Referrer-Policy: strict-origin, X-Frame-Options: DENY, and COOP: same-origin to dynamic routes — wrapped so the headers ship even when a downstream handler throws a 500.

Classes: .segmented · .accordion / .accordion-item · .dropzone / .dz-files in components.css §10–§12.
